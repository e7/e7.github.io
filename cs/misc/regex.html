    # 限定符

    限定符 说明
    ？ 表明前面的字符需要出现0次或1次，即可有可无
    * 用于匹配0个或多个字符，也是可有可无但不针对它前面的字符
    + 表明前面的字符需要出现至少1次
    {...} "+"限定符的精确版，比如{6}匹配前面字符出现6次，{2,6}匹配前面的字符出现2到6的任意次，{2,}匹配前面的字符至少2次

    TIP

    如果需要匹配的不是前面1个字符而是多个，可以用小括号括起来，比如“(ab)?”。

    如果需要限定匹配的字符集合，可以用方括号，比如“[abc]+”，表示加号前面的字符出现至少1次，但只能是这3个中的，如果字符比较多，可以用简写形式，比如"[A-Z0-9]+"，“^”可以取反，比如[^0-9]表示不能取数字字符。

    # 元字符

    大多数元字符都是以反斜杠"\"开头，纯粹是为了方便。

    元字符 说明
    \d 匹配数字字符，等价于[0-9]
    \w 单词字符，英文字符+数字+下划线的组合
    \s 空白符，包含tab、换行及空格
    \D 非数字字符
    \W 非单词字符
    \S 非空白符
    . 任意字符，除了换行符
    ^ 匹配行首
    $ 匹配行尾
    \b 匹配单词边界

    # 或运算

    (a|b) # 匹配a或b
(ab)|(cd) # 匹配ab或cd

    # 贪婪与懒惰匹配

    默认的匹配模式为贪婪匹配，也就是尽可能多的匹配字符。在限定符之后加上“?”可切换为懒惰匹配，即满足正则的条件下尽可能少匹配。

    示例：

    abccbaa1111123

# ------- 分割线 -------
aa1+    # 这会匹配到最后一个1
aa1+?   # 这只会匹配到第一个1
