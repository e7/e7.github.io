# Vcpkg

vcpkg默认安装x86的库，如果在CMake中配置的是64位的编译器，则会报错，找不到库。

两种解决方式：

  1. 安装库的时候指定为x64

    vcpkg install zlib:x64-windows
vcpkg install zlib --triplet x64-windows

  2. 设置环境变量（记得重启以使变量生效）

    VCPKG_DEFAULT_TRIPLET=x64-windows

使用多线程库在windows平台下可能出现链接错误：

  • win下链接有/MT(d)和/MD(d)的C运行时之分，可以在CMakefileLists.txt中设置

    list(APPEND CMAKE_PREFIX_PATH "$ENV{VCPKG_ROOT}/installed/x64-windows-static-md")
set(VCPKG_TARGET_TRIPLET x64-windows-static-md) # 默认是MT，这里的设置跟安装时候的要匹配

    更细节的可以使用如下环境变量控制：

    # VCPKG_LIBRARY_LINKAGE Determines if a library/port is build as a static or dynamic library
# VCPKG_CRT_LINKAGE Determines if the static (/MT(d)) or dynamic (/MD(d)) CRT (C runtime) is used

x64-windows:
VCPKG_LIBRARY_LINKAGE = dynamic
VCPKG_CRT_LINKAGE = dynamic

x64-windows-static:
VCPKG_LIBRARY_LINKAGE = static
VCPKG_CRT_LINKAGE = static

x64-windows-static-md:
VCPKG_LIBRARY_LINKAGE = static
VCPKG_CRT_LINKAGE = dynamic

  • 命令行

    cmake -S. -B build -DCMAKE_TOOLCHAIN_FILE=${vcpkgRoot}/scripts/buildsystem/vcpkg.cmake

  • vscode

    settings.json配置

    {
    "cmake.configureSettings": {
        "CMAKE_TOOLCHAIN_FILE": "/xxxx/scripts/buildsystem/vcpkg.cmake"
    }
}

# Tips

  • 收集代码文件

    file(GLOB SRC_FILES "${PROJECT_SOURCE_DIR}/src/*.cpp" "${PROJECT_SOURCE_DIR}/src/*.h")

  • 为目标添加编译选项

    target_compile_features(${CMAKE_PROJECT_NAME} PRIVATE cxx_std_17)

  • msvc编译器默认不识别不带bom的utf8编码的源码文件，需要手动指定编码类型

    # 注意要放在add_executable指令之前才能起作用
IF (WIN32)
  add_compile_options("$<$<C_COMPILER_ID:MSVC>:/source-charset:utf-8>")
  add_compile_options("$<$<CXX_COMPILER_ID:MSVC>:/source-charset:utf-8>")
ENDIF()

  • windows下命令行编译指令

    cmake -G "Visual Studio 16 2019" -A x64 -S . -B build64
cmake --build build64 --config Release
