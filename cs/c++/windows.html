# Tips

  • 为何高精度计时器不准确？

    QueryPerformanceCounter 可以获取性能计数器的当前值，单位是赫兹，根据QueryPerformanceFrequency返回的频率信息可以实现高精度计时，但在多核下是不准确的，在某一线程内调用QueryPerformanceCounte，线程会切换于不同的核心之间，该函数内部又没有加锁，这时候QueryPerformanceCounter返回值可能是不正确的，即计时器并不能保证是顺序累加的，可以通过绑定线程的CPU亲缘性解决：

    HANDLE thread = GetCurrentThread();
 
// Set affinity to the first core
DWORD_PTR oldMask = SetThreadAffinityMask(thread, GTimerMask );
 
// Query the timer
QueryPerformanceCounter(&mStartTime);
mStartTick = GetTickCount();
 
// Reset affinity
SetThreadAffinityMask(thread, oldMask);

  • SetWindowHookEx的钩子回调为何很慢？

    消息钩子依赖windows消息循环，如果程序中没有线程处理事件消息就会很慢，具体原因未知，解决方案是增加消息事件循环：

    MSG nMsg {};
while (::GetMessage(&nMsg, nullptr, 0, 0)) {
    ::TranslateMessage(&nMsg);
    ::DispatchMessage(&nMsg);
}
