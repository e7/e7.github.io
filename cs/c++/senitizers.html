# google sanitizers

google的一系列Sanitizer (opens new window)，可以用于定位程序中的系列问题，常用的Sanitizer包括

  • Address Sanitizer（ASan）：用于检测内存使用错误
  • Leak Sanitizer（LSan）：用于检测内存泄漏
  • Thread Sanitizer（TSan）：用于检测多线程间的数据竞争和死锁
  • Memory Sanitizer（MSan）：用于检测使用未初始化内存的行为

# ASAN分析内存问题

# 安装

# centos
yum install libasan -y
# 需要llvm-symbolizer做符号解析
yum install llvm -y

# 使用

  • 编译链接选项

    # cmake
add_compile_options(-O0 -ggdb -std=c++17 -Wall -Wextra -mavx2
    -fsanitize=address
    -fno-omit-frame-pointer
    -fno-optimize-sibling-calls
    -fsanitize-address-use-after-scope
    -fsanitize-recover=address)
add_link_options(-fsanitize=address)

  • 环境变量

    # llvm-symbolizer的运行路径
export ASAN_SYMBOLIZER_PATH="/usr/bin/llvm-symbolizer"
export ASAN_OPTIONS="halt_on_error=0:log_path=xxx/asan.log:detect_stack_use_after_return=1"
    flag 含意
    halt_on_error 默认为1，ASan检测到错误后会中止程序设为0后，ASan检测到错误后不会中止程序
    log_path 设置错误信息的输出路径
    detect_stack_use_after_return 是否检测use after return错误默认为0，即关闭该检测功能

    更多请参考这里 (opens new window)

  • 注意事项

    • asan版本程序在Linux环境下运行时会额外申请20TB的虚拟内存（这也可以作为检验ASan是否工作的标志）
      • 需要确保/proc/sys/vm/overcommit_memory的值不为2
    • asan版本性能大幅受损，大约会下降2倍左右
    • asan工具不是万能的，必须要跑到有问题的代码才可以暴漏出来
