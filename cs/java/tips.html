# Java

  • 重写了equals方法必须也重写hashcode方法

    默认情况下，equals方法是比较两个对象的地址，如果相同则表示是同一个对象，那么hashcode也一定是一样的。

    一些容器比如HashMap使用了hashcode作索引key，如果重写了equals方法而不重写hashcode则可能会破坏二者的一致性，这样可能导致HashMap出现重复元素（即equals看是一样的，hashcode又不一样）或者不同的元素却冲突。

  • Optional

    • 勿使用null判空，容易遗漏，ifPresent()都比它强，因为Optional对象不get()就用不了，从而提醒你判空。
    • 更好的的方式是使用orElseThrow()或者其它的orElseXxx()。
    • Optional应该仅用在函数返回值中，出现在其它的任何地方（比如入参，类属性等）都是不推荐的。
    • map()可以方便地取得对象内部的属性值而不必关系Optional是否有值，还可以多级调用取得更深的值。

  • 不要用isXXX定义布尔值，因为不同的库序列化可能会不一样导致跟预期不符。

  • 为什么java程序需要预热？

    java是半编译半解释型语言，jvm中有jit编译器，需要运行一段时间后才能找出热点代码，进而编译这部分代码为机器指令并缓存，这个过程叫预热，纯编译型语言是没有这个过程的。

# Springboot

  • 配置文件外置

    # spring.cloud.bootstrap.location
# spring.config.local
java $JAVA_OPTS -jar xxx.jar --spring.cloud.bootstrap.location=./bootstrap.yml --spring.config.local=./application.yml

  • 优雅停机

    server:
  # 2.3.0之后的特性
  shutdown: graceful

spring:
  main:
    # springboot注册的jvm停机钩子，开启才能优雅停机，默认开启
    register-shutdown-hook: true

  • 注解

    • @PreDestroy bean销毁前执行

  • 自定义配置的idea自动补全

    resources目录下创建META-INF目录，再在META-INF目录下创建additional-spring-configuration-metadata.json文件，示例如下：

    {
  "properties": [
    {
      "name": "config1.a",
      "type": "java.lang.String",
      "description": "Description for config1.a."
    },
    {
      "name": "config1.b",
      "type": "java.lang.String",
      "description": "Description for config1.b."
    }
}

    然后重新编译，一定要编译不然不能生效。

  • consul

    • 如果依赖了spring-cloud-starter-consul-all或spring-cloud-starter-consul-config，需要将相关配置放到bootstrap.yml中而不是application.yml。
    • 如果使用了配置功能，那么key中不能有“-”，否则无法加载配置。

# Mybatis-Plus

  • 实体模型中不要使用原始类型，因为原始类型有零值而不是null，所以更新的时候字段会被写入零值。
  • 如果使用了数据库自增字段作为id，需要增加注解
    @TableId(type = IdType.AUTO)
