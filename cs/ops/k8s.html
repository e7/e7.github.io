# Docker

  • docker使用CGroups（control groups）实现资源的配额管理，可限制CPU、内存、网络等。

    • 利用cgroup限制cpu示例

      cd /sys/fs/cgroup/cpu
mkdir my_limit # 创建后系统会自动在该目录创建一堆文件
echo 20000 > /sys/fs/cgroup/cpu/my_limit/cpu.cfs_quota_us # 最多能使用20%的时间片
echo 12345 >> /sys/fs/cgroup/cpu/my_limit/tasks # 被限制的进程列表，一行一个

# docker版
docker run -d --cpu=".5" nginx # 限制cpu为50%

    • 镜像存储

      • 基于checksum的去重存储，比如底层的使用同一版本的操作系统，则只会存储一份，不像虚拟机有多份系统。

# K8S

  • 优点：

    • 按需水平和垂直扩容
    • 自动副本控制
    • 服务发现和路由
    • 自动部署和回滚

  • 组件

    • master

      • kube-api-server：负责提供k8s的api服务，非常关键，worker节点与master节点通信的桥梁
      • etcd：kv数据库，配置信息等的存储，保存k8s集群信息
      • kube-scheduler：监视那些新创建的未指定运行节点的pod，并依据一定的调度策略选择节点让pod在其上运行
      • kube-controller-manager：一堆控制器的聚合体，为了降低复杂度各种控制器被编译到了一起
        • 节点控制器：负责节点在出现故障时通知和响应
        • 副本控制器：负责为系统中的每个副本控制器对象维护正确数量的pod
        • 终端控制器：负责填充终端对象（Endpoints），即加入service和pod
        • 服务账户和令牌控制器：为新的命名空间创建默认账户和API访问令牌

    • slave

      • kubelet：类似一个agent，处理来着master的请求
      • kube-proxy：集群中每个节点的网络代理，是实现service概念的组成部分，维护了节点上的网络规则（iptables）
      • container-runtime：负责运行容器的软件，支持docker、containerd、cri-o、rktlet及任何Kubernetes CRI的实现

    • 插件

      • DNS：虽然是插件，但基本上是必需的
      • Web界面（dashboard）：仪表盘，可以集中管理集群中的应用程序，也用于集群本身故障的排查
      • 资源监控（metric-server）：将关于容器的一些常见的时间序列度量值保存在一个集中的数据库中，并提供用于浏览这些数据的界面
      • 集群层面日志：负责将容器的日志数据保存到一个集中的日志存储中，该存储能提供搜索和浏览接口

# Minikube

为何需要？

搭建k8s集群需要比较多的资源，步骤也相对繁琐，而minikube在保证兼容的基础上简化搭建步骤，可单机部署且占用资源低，适合开发环境

# 安装（Linux环境）

  1. 下载安装kubectl
    curl -LO "https://dl.k8s.io"/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl
sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
  2. 下载安装minikube
    # 假定你已经安装好了docker
# 安装过程中需要访问gcr.io等境外域名，请确保你的网络通畅，或者使用国内镜像地址
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube
minikube start
  3. 启动dashboard
    minikube dashboard

# 启动后会尝试打开浏览器
# 因为监听地址在127.0.0.1，如果需要远程访问还需要启动代理，之后将url中的ip端口改为代理的即可
kubectl proxy --address='0.0.0.0' --disable-filter=true
