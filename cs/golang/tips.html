#开源

  • 交叉编译

    GOOS=windows GOARCH=amd64 CGO_ENABLED=1 CC="x86_64-w64-mingw32-gcc" CGO_CFLAGS="-DSQLITE_ENABLE_DBSTAT_VTAB -DSQLITE_DEFAULT_PAGE_SIZE=512" go build -o demo.exe -ldflags '-w -s' cmd/main.go

  • 环境变量

    • GOTRACEBACK，控制panic时的行为
      1. none，不输出调用栈
      2. single（默认值），输出导致异常的协程调用栈
      3. all，输出用户创建的所有协程调用栈
      4. system，类似all，但还包括GO框架底层实现创建的协程
      5. crash，输出所有的协程调用栈，并生成core

  • 逃逸分析

    • 原则1：指向栈上对象的指针不能被存储到堆中

    • 原则2：指向栈上对象的指针不能超过该栈对象的生命周期（即如果超过，则将该栈对象分配到堆中）

    • 输出逃逸分析信息，编译时加入-m=2

      go tool compile -m=2 test.go

  • 终端下执行单元测试

    # 单个函数
go test -run Test_Func1 -v Awesome{,_test}.go
# 执行所有
go test ./pkg/...

  • 验证某个struct是否实现了某个接口，没（完全）实现就会编译报错

    var (
  // 匿名变量，未实现则赋值会报错
  _ Render = &JSON{}
)

  • return不是原子的，先将返回值保存到栈上，然后执行defer，再返回

  • 切片拆分

    // SplitSlice 切片拆分
func SplitSlice[T any](dataList []T, perBatch int) [][]T {
	nBatch := (len(dataList) + perBatch - 1) / perBatch
	idsList := make([][]T, nBatch-1, nBatch)
	for i := 0; i < nBatch-1; i++ {
		idsList[i] = dataList[perBatch*i : perBatch*(i+1)]
	}
	return append(idsList, dataList[perBatch*(nBatch-1):])
}

  • runtime.MemStats

    Alloc: 已分配的堆对象内存大小，即已分配但未使用的内存大小。 TotalAlloc: 已分配的总内存大小，包括已释放的内存。 Sys: 运行时系统从操作系统申请的内存大小。 Lookups: 已进行的指针查找次数。 Mallocs: 分配的堆对象数量。 Frees: 释放的堆对象数量。 HeapAlloc: 当前堆内存的使用量。 HeapSys: 已经向操作系统申请的堆内存大小。 HeapIdle: 未被使用的堆内存大小。 HeapInuse: 已被使用的堆内存大小。 HeapReleased: 从操作系统归还的堆内存大小。 HeapObjects: 当前堆中对象的数量。 StackInuse: 当前栈内存的使用量。 StackSys: 运行时系统从操作系统申请的栈内存大小。 MSpanInuse: 当前 mspan 结构体（管理一段内存区域的结构体）的使用量。 MSpanSys: 运行时系统从操作系统申请的 mspan 结构体的大小。 MCacheInuse: 当前 mcache 结构体（线程缓存，用于存储本地对象）的使用量。 MCacheSys: 运行时系统从操作系统申请的 mcache 结构体的大小。 BuckHashSys: 分配和管理 buckhash 的堆内存大小。 GCSys: 垃圾回收元数据的内存大小。 OtherSys: 其他运行时系统使用的内存大小。

  • 避坑

    • 闭包陷阱

      var data = []int{1,2,3}
for _, d := range data {
  // d虽然是int类型（值为data元素的一份拷贝），但因为调用后又继续赋值（即使只循环一次也是），
  // 所以闭包会以【引用】捕获变量d，goroutine执行时可能已经循环完毕，所以总是输出最后一个元素
  go func() {
    fmt.Println(d)
  }()
}

    • SetDeadline和SetXxxDeadline是设置的未来的某个时刻，所以每次读写之前需要重新设置

    • json解析尽量加注解，防止匿名嵌套定义时无法解析出正确的值

      type Test struct {
    Value struct {
        X int `json:"x"`
        Y string `json:"y"`
    } `json:"value"`
}
